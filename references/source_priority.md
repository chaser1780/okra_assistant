# 数据源优先级

本文定义 `okra assistant` 当前对主要数据源的使用优先级、降级策略和落盘位置。

## 1. 总原则

1. **官方日频数据优先于盘中估算数据**
2. **结构化 API 优先于页面抓取**
3. **可追溯落盘优先于临时内存结果**
4. **同一类数据必须有明确主源与降级源**

## 2. 行情与净值

### 2.1 官方净值

- 主源：Eastmoney 历史净值接口
- 用途：
  - 组合每日官方净值重估
  - 夜间复盘真实涨跌判断
  - 组合长期收益与资产基线
- 当前实现：
  - `https://api.fund.eastmoney.com/f10/lsjz`

### 2.2 实时估值

- 主源：`fundgz.1234567.com.cn`
- 用途：
  - 日内盘中实时收益估算
  - 日内战术方向辅助判断
- 注意：
  - 不等同于官方净值
  - QDII 等类别存在明显时差与滞后

### 2.3 盘中代理行情

- 主源：Sina HQ ETF / 指数代理
- 用途：
  - 对无实时估值或低置信估值进行补充
  - 作为日内方向验证源
- 注意：
  - 它是代理，不是基金真实净值

## 3. 新闻与事件

### 3.1 基金公告

- 主源：Eastmoney 基金公告接口
- 用途：
  - 公告、基金经理变动、定期报告等高相关信息

### 3.2 基金页面文章

- 主源：Eastmoney 基金页面文章
- 用途：
  - 获取补充性资讯与主题新闻

### 3.3 新闻使用原则

- 公告优先于普通资讯
- 新增信息优先于旧闻重复
- 能映射到具体基金优先于泛行业表述
- 所有新闻都必须落盘并保留 URL

## 4. 模型层输入优先级

在同一只基金的日内判断中，建议按以下顺序理解数据：

1. 官方净值日期与组合估值日期
2. 盘中实时估值及其置信度
3. 盘中代理行情
4. 最新相关新闻与公告
5. 复盘记忆与偏置调整

## 5. 降级策略

### 5.1 官方净值

- 若目标日期无官方净值：
  - 回退到最近交易日
  - 明确标记 `date_match_type`
  - 不把“上一交易日”误当作“当日官方净值”

### 5.2 实时估值

- 若估值跨日、置信度不足或类别不适用：
  - 按 `realtime_valuation.toml` 规则退回代理行情

### 5.3 盘中代理行情

- 若抓取失败：
  - 可回退到旧快照
  - 但必须标记 stale/fallback

### 5.4 新闻

- 若新闻抓取失败：
  - 写入占位记录
  - 不可静默吞掉

## 6. 落盘位置

- 官方净值：
  - `raw/quotes/<date>.json`
- 新闻：
  - `raw/news/<date>.json`
- 盘中代理：
  - `db/intraday_proxies/<date>.json`
- 实时估值：
  - `db/estimated_nav/<date>.json`
- 实时收益快照：
  - `db/realtime_monitor/<date>.json`
- 官方净值重估：
  - `db/portfolio_valuation/<date>.json`

## 7. 当前说明

当前项目已做到：

- 官方净值、实时估值、代理行情三者分开处理
- 每一层都带时间与新鲜度语义
- 官方净值与日内估值不再混为一个口径

后续若增加新源，应优先遵守这份文档中的优先级与降级规则，而不是直接在业务脚本中插入新的抓取逻辑。
