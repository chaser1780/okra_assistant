# 基金研究桌面应用 PRD（第一版）

- 版本：v1
- 更新日期：2026-03-09
- 目标形态：本地桌面应用
- 推荐框架：`PySide6`

## 1. 产品目标

把当前基于脚本的基金研究系统，包装成“打开即可用”的本地研究工作台。

第一版不追求复杂账号系统，不追求云端同步，不追求自动下单，只聚焦：

- 查看今日研究结论
- 查看多智能体输出
- 查看复盘与记忆
- 录入交易并回写持仓
- 手动触发日内/夜间任务

## 2. 设计原则

- 研究优先，不做资讯门户。
- 首页默认回答“今天最该看什么”。
- 所有数据继续以 `F:\okra_assistant` 为主。
- 尽量不用重型浏览器内核，减少 `C:` 额外占用。
- 第一版以只读 + 轻操作为主，不急于做复杂交互。

## 3. 推荐技术方案

### 方案结论

- 首选：`PySide6`
- 暂不建议：`Electron`
- 当前执行策略：桌面端统一为 `PySide6 / Qt` 单栈，Tk 旧实现已从主代码路径移除

### 原因

- 当前引擎已经是 Python 脚本，直接复用最顺。
- 可以直接读取 JSON / Markdown / 日志。
- 对文件路径、任务调度、缓存位置更可控。
- 可减少额外 Chromium 体积与缓存。

## 4. 页面结构

### 4.1 Dashboard

目标：

- 打开后第一眼看到今日状态

显示内容：

- 今日 14:00 报告是否已生成
- 今日 21:00 复盘是否已生成
- 上次运行时间
- 今日建议动作数量
- 市场状态摘要
- 现金仓可调拨金额
- 失败 agent 提示

### 4.2 Today Research

目标：

- 看今天每只基金的研究结论与动作建议

显示内容：

- 左侧基金列表
- 中间基金详情
- 右侧最终建议

字段：

- 当前持仓金额
- 持有收益率
- 盘中代理变化
- 实时估值变化
- 新闻摘要
- 最终动作
- 建议金额
- 支持该建议的 agents

### 4.3 Agents Room

目标：

- 看每个 agent 怎么想

显示内容：

- 每个 agent 卡片
- 研究经理与风控经理单独高亮
- 分歧面板
- 失败面板

### 4.4 Portfolio

目标：

- 看组合结构与资金状态

显示内容：

- 总资产
- 总盈亏
- 角色分布
- 主题暴露
- 单只基金上限占用
- 现金仓余额

### 4.5 Trade Console

目标：

- 手工录入成交并形成闭环

显示内容：

- 买入/卖出/换基录入
- 当日已执行动作
- 历史交易流水
- 执行状态标记

### 4.6 Review & Memory

目标：

- 看系统是否在进化

显示内容：

- 夜间复盘报告
- lessons
- bias adjustments
- agent feedback
- 最近哪些建议最有效/最无效

### 4.7 Settings & Data

目标：

- 看配置、路径、任务与日志

显示内容：

- 模型配置状态
- 数据源状态
- 计划任务状态
- 报告目录
- 日志目录
- 手动重跑按钮

## 5. 交互优先级

第一版优先做：

1. Dashboard
2. Today Research
3. Agents Room
4. Review & Memory
5. Settings & Data

第二版再做：

1. Trade Console
2. 组合图表
3. 执行状态流转

## 6. 文件与缓存策略

- 应用配置：优先放 `F:\okra_assistant\app`
- 应用日志：放 `F:\okra_assistant\logs\desktop`
- 临时目录：放 `F:\okra_assistant\temp`
- 图表缓存：放 `F:\okra_assistant\cache`
- 不主动在 `C:` 保存数据库、图片缓存或浏览器缓存

## 7. 第一版实施建议

### Step 1

做只读桌面壳：

- 读取报告
- 读取 agent 输出
- 展示首页和建议页

### Step 2

增加运行按钮：

- 触发日内报告
- 触发夜间复盘

### Step 3

增加交易录入：

- 调 `record_trade.py`
- 自动回写持仓

### Step 4

增加图表与统计：

- 复盘命中率
- agent 偏差趋势

## 8. 当前不做

- 自动下单
- 微信推送
- 多用户登录
- 复杂数据库
- 云端同步
